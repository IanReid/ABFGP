BiRC Python GFF library.

                                                                                          Version 1.0
 

                                                    Class : Feature

The feature class encapsulates one line of a gff file. Parameters specified with brackets as
in [strand], are optional parameters. Names in italics will refer to parameter values.
 

Constructors :

    Feature(line) :
        Constructs a new feature object from the string line. The various fields in line should be separated by tab characters.
        An FeatureInputError exception is raised if the line is invalid.

    Feature():
        Creates an empty feature object (with default values)

Access methods :


    seqname([name]) :
        Returns the sequence name of the feature. If name is specified then seqname will be assigned this value.

    source([name]) :
        Returns the source name of the feature (e.g.. "genbank") .If name is specified then source will be assigned this value.

    feature([name]) :
        Returns the feature name (e.g.. "gene"). If name is specified then source will be assigned this value.

    start([value]) :
        Returns the start position of a feature. If value is specified then start will be assigned this value.

    end([value]) :
        Returns the end position of the feature. Always the case that end() >= start().
        If value is specified then end will be assigned this value.

    score([value]) :
        Returns the score of this feature.
        If value is specified then score will be assigned this value.

    strand([value]) :
        Returns the strand of this feature (e.g.. "+","-" or ".")
         If value is specified then strand will be assigned this value.

    frame([value]) :
        Returns the reading frame of this feature ("0","1","2" or ".")
        If value is specified then frame will be assigned this value.

    attribute([value]) :
        Returns the attribute of this feature (an optional string)
        If value is specified then attribute will be assigned this value.

Methods :


    len (feature) :
        Returns the lenght of the feature - defined as : feature.end() - feature.start() + 1.

    cmp(f2) :
        Compares the feature (f1) with feature f2 according to the lexicographical ordering of (start,end) coordinates.
        returns -1 if f1<f2, 0 if f1==f2 and 1 if f1>f2. This method is used by the sort method in the GFF class.
        Operators such as : <, >, <=, =>, ==, can be used on two feature objects.
        NB : comparisons are not strand sensitive.

    copy() :
        Returns a copy of this feature.

    remap(offset) :
        Adds offset ot the features start and end coordinate - i.e. shifting the feature.

    overlap(feature2,[strand]) :
        Checks for overlaps between the feature and feature2. If strand is 1 then strands will be taken into
        consideration when calculating the overlap. Returns 0 if no overlap otherwise returns the size of
        the overlap. Overlap between the two features (f1 and f2) is defined to be when one of the following
        conditions are true :
        1. f2.start() <= f1.end() <= f2.end()
        2. f1.start() <= f2.end() <= f1.end()

    merge(feature2,[strand]) :
        If the two features overlaps (as specified in the overlap function) and taking strand into account,
        then a merged feature (i.e. coordinates 10-30, and 20-40 will result in 10-40) will be returned.
        Returns a new feature, where the other attributes of it will be the same as in the calling feature object.
        If no overlap then it will just return None (null pointer).

    intersect(feature2,[strand]) :
        If the two features overlaps (as specified in the overlap function) taking strand into account,
        then the intersection of the calling feature and the other will be returned. If no intersection between the two
        features exists then none (null pointer) will be returned. The intersection of for example two
        features with coordinates 10-30, and 20-40 wil result in a new feature with the coordinates 20-30 and all other
        attributes will be the same as the calling features attributes.
